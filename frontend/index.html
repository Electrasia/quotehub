🔌 LLM Setup

Connection settings for the AI language model used for extraction and OCR.

📦 Extraction

When enabled, the system automatically selects the best method: Vision LLM for scanned PDFs, Text LLM for documents with extractable text. Falls back to local rules if the AI is unavailable.

OCR (scanned PDFs)

Scanned/image-only PDFs have no text for the local extractor to read. Enable OCR to run pytesseract on each page. If tesseract's quality is low (avg confidence < 40% or fewer than 5 numbers detected), a vision LLM can be used as a second pass.

👥 Users Management

QuoteHub – Quick User Guide

📖 Purpose

QuoteHub is an AI-powered quotation database. Upload PDF quotations, let the AI extract the data automatically, review and edit, then store everything for fast search and reuse.

⬆️ Upload Documents

  1. Go to Process
  2. Drag & drop or select one or more PDF or XLSX files
  3. Click Upload

📋 File Queue

  • Review the list of uploaded files
  • Remove any unwanted files
  • Click Process All to start

⚙️ Processing

  • The AI extracts data from each page/sheet automatically
  • PDF and XLSX files are both supported
  • Wait for the current file to complete (or click Cancel to stop)

✏️ Review & Edit

Verify each field before saving:

  • Brand, Model, Description, Price, Supplier, Date
  • Edit any incorrect or missing values directly in the table
  • Use Bulk Apply for repeated values (e.g. same Brand across many rows)

💾 Save Data

  • Click Confirm & Save to store the data
  • The quotation is then searchable for future use

🔍 Search & Reuse

  • Go to Search
  • Type a supplier, date, or item description
  • Browse, filter, and reuse past quotations

🧹 System Cleanup (Master Only)

Permanently delete old quotations and (optionally) their files.

  • Set a time period (in months)
  • Click Preview first to see what will be deleted
  • Optionally enable file deletion
  • Type the word DELETE to confirm

⚠ Warning: This action is irreversible. Once deleted, data cannot be recovered.

🔄 Automatic Backups

The system automatically creates daily encrypted backups at 03:00. Weekly snapshots are promoted on Sundays. Event backups are taken before imports and app updates.

  • Settings → Backup / Restore shows the last and next backup time
  • Click Restore from automatic backup to browse and restore a previous snapshot
  • Retention: 7 daily, 4 weekly, 45 days for event backups
  • Automatic backups are encrypted with a machine-bound internal key (no password needed to restore)

👤 User Roles

  • User — search and browse existing data
  • Admin — upload, process, edit, export backup, view logs, read AI settings (no changes). Cannot access General, AI Settings, Import, Cleanup, User Management, or restore automatic backups
  • Master — full control, including all settings, import/export, cleanup, user management, and backup restore

⚠️ Important Notes

  • Always review extracted data before saving
  • System Cleanup is permanent and cannot be undone
  • Processing time depends on file size and number of pages
  • Passwords must be at least 12 characters — use the strength meter as a guide
  • Exported backups are encrypted per-file. The password is never stored. If you lose it, the backup is permanently unrecoverable
  • Export and Import are master-only operations
